## 摘要
随着模型参数和训练数据的增加，大型视觉语言模型（LVLMs）的指令遵循和泛化能力得到了显著提升。基于混合专家（MoE）架构，LVLMs在保持推理成本的同时扩展了参数容量。然而，传统的MoE方法采用Top-k静态路由策略，未能考虑输入的变化并自适应地选择专家数量，导致资源利用次优。在本文中，我们提出将令牌路由视为信息编码任务，将动态路由建模为编码中的最小描述长度（MDL）问题。通过验证MoE场景中MDL与门控熵之间的联系，我们引入了基于门控熵的不确定性感知自适应路由（GeMoE）。与传统的静态或基于启发式的动态路由方法不同，GeMoE显式地建模了模型复杂度与性能之间的权衡。通过使用门控熵评估令牌的复杂度，GeMoE自适应地确定每个令牌应参与的专家数量。在广泛的主干网络和基准测试中，与原始静态路由相比，我们的方法平均保留了99.5%的性能，同时将平均专家激活稀疏性提高了36.5%。

## Abstract
With the increase in model parameters and training data, the instruction following and generalization capabilities of Large VisionLanguage Models (LVLMs) have been significantly improved. Based on the Mixture of Experts (MoE) architecture, LVLMs expand their parameter capacity while maintaining the inference cost. However, traditional MoE methods employ a Top-k static routing strategy, which fails to account for variations in the input and adaptively select the number of experts, resulting in suboptimal resource utilization. In this paper, we propose viewing token routing as an information encoding task, framing dynamic routing as a Minimum Description Length (MDL) problem in encoding By validating the connection between MDL and gating entropy in the MoE scenario, we introduce Gating Entropy-based Uncertainty-aware Adaptive Routing (GeMoE) for MoE. Unlike traditional static or heuristic-based dynamic routing methods, GeMoE explicitly models the trade-off between model complexity and performance. By using gating entropy to assess the complexity of tokens, GeMoE adaptively determines the number of experts each token should engage. On a wide range of backbones and benchmarks, our method achieves 99.5% average performance retention compared to the original static routing, while improving average expert activation sparsity by 36.5%.

- **研究动机**：随着大型视觉语言模型（LVLM）的规模不断扩大，其推理成本高昂。混合专家（MoE）架构通过在推理时仅激活部分专家，能够在不显著增加计算量的情况下扩展参数容量。然而，传统的MoE使用固定的 **Top‑k 静态路由**，为每个令牌分配相同数量的专家，忽略了令牌间的复杂度差异：简单令牌浪费资源，复杂令牌则可能因专家不足而丢失语义信息。这导致资源利用次优且模型性能无法充分发挥。

- **整体含义**：本文提出应将令牌路由视为信息编码任务，动态路由本质上是**最小描述长度（MDL）**问题——在模型复杂度与数据拟合之间寻找最优平衡。作者通过建立MDL与门控熵（Gating Entropy）之间的联系，提出一种基于门控熵的不确定性感知自适应路由方法 **GeMoE**，实现了“令牌越不确定（熵越高），分配越多专家”的动态分配策略。

### 2. 论文提出的方法论

- **核心思想**：将动态路由形式化为MDL最小化问题。MDL = 模型编码长度 + 数据拟合残差编码长度。增加专家可降低拟合残差，但会增加模型复杂度。通过信息增益（Information Gain）与门控熵的关系，发现**高门控熵的令牌在增加专家时能带来更大的信息增益**，从而降低MDL。因此，门控熵可作为自适应调节专家数量的代理指标。

- **关键技术细节**：
  - **门控熵计算**：对于令牌 \(x\)，利用路由器 \(R\) 输出的路由分布 \(R_{\text{norm}}(x) = [w_1, w_2, \ldots, w_K]\)，计算熵 \(H(x) = -\sum w_i \log_2 w_i\)。
  - **专家分配预测器（EAP）**：一个轻量线性层，输入令牌特征，输出专家数量区间 \([k_{\text{low}}, k_{\text{high}}]\) 上的概率分布，通过期望值 \(k_{\text{soft}}\) 并四舍五入得到最终整数 \(k\)。在本文中 \(k_{\text{low}}=1, k_{\text{high}}=8\)（对MolmoE‑1B‑7B）或 \(k_{\text{high}}=6\)（对DeepSeek‑VL2‑Tiny）。
  - **单调性损失（\(\mathcal{L}_{\text{mono}}\)）**：强制批内任意令牌对 \((x_i, x_j)\)，若 \(H(x_i) > H(x_j)\)，则 \(k_{\text{soft}}^i > k_{\text{soft}}^j\)（加上一个边界margin）。这保证了门控熵与分配专家数量的正相关性。
  - **总体损失**：\(\mathcal{L} = \mathcal{L}_{\text{ce}} + \alpha \mathcal{L}_{\text{mono}} + \beta \mathcal{L}_{\text{lb}}\)，其中 \(\mathcal{L}_{\text{lb}}\) 为负载均衡损失（平衡各专家接收令牌的比例）。

- **算法流程简述**：
  - 对每个令牌，先计算门控熵。
  - 利用EAP预测所需专家数 \(k\)。
  - 在Top‑k选中的专家上执行加权求和输出。
  - 训练时额外计算单调性损失，反向传播更新EAP及路由参数。

### 6. 论文的主要结论与发现

- **核心结论**：门控熵可作为MDL最小化的有效代理，高熵令牌增加专家能带来更大信息增益，从而降低MDL。基于此设计的GeMoE在保持99.5%平均性能的同时，将专家激活稀疏性平均提升36.5%。
- **关键发现**：

### 7. 优点

- **理论创新**：首次将动态路由显式建模为MDL问题，并建立其与门控熵之间的严格推导，提供了坚实的理论依据，而非纯启发式。
- **轻量高效**：EAP仅为额外线性层，计算开销可忽略，内存占用与原模型相同；且推理时可直接根据熵预测专家数，无需反复计算信息增益。
- **通用性强**：在两种不同规模的MoE LVLM（MolmoE‑1B‑7B、DeepSeek‑VL2‑Tiny）以及纯语言MoE模型（OLMoE‑1B‑7B）上均取得优于静态Top‑k的稀疏‑性能权衡。
- **实验充分**：不仅覆盖大量VL benchmark，还扩展到LLM领域，并进行了详细的消融和可视化分析。

### 8. 不足与局限

- **实验覆盖**：仅在1‑7B级别的小规模MoE模型上验证，未在更大规模（如30B+）或真实部署场景中测试，泛化性需进一步确认。
- **训练数据限制**：微调仅使用LLaVA‑1.5‑558k（558k样本），训练1 epoch，可能未能充分学习复杂的专家分配模式；理论上更长的训练或更多样化的数据可能进一步提升性能。
- **动态路由的副作用**：EAP的预测依赖门控熵，若路由器本身不够鲁棒（如存在负载不均衡），则熵的可靠性可能下降；虽然论文使用了负载均衡损失，但在极端不平衡下仍可能失效。
- **应用限制**：方法要求对路由器输出计算门控熵，仅适用于基于软路由的MoE；对于硬路由（如只选top‑1且无权重）的变体，无法直接移植。
- **对比基线**：部分基线（如AdaMoE、MoE++）在原生MoE上表现较差，可能因为其设计针对“复制专家”场景，而原生MoE专家已具有多样性，导致泛化不佳。对比结果存在一定先天劣势。
