## 摘要
使用低秩自适应（LoRA）在终端用户数据上微调大型语言模型（LLM）可在保护数据隐私的同时提供个性化体验，但在消费级硬件上面临严重的内存限制。微调过程中的峰值内存通常超出设备限制，尤其是对于具有数十亿参数和长上下文训练数据的模型。本文介绍了一套互补技术，可在不牺牲模型质量的情况下减少内存占用：（1）基础模型量化与即时反量化，（2）结合选择性激活缓存和磁盘卸载的内存高效检查点技术，（3）使用语义相关令牌子集的softmax近似，以及（4）logits掩码。在Llama-3.2 3B和Qwen-2.5 3B上的实验表明，峰值内存减少了高达26倍和28倍，从而在资源受限的设备上实现微调。

## Abstract
Fine-tuning of Large Language Models (LLMs) using Low-Rank Adaptation (LoRA) on an end-user's data offers personalized experiences while keeping data private, but faces severe memory constraints on consumer hardware. Peak memory during fine-tuning often exceeds device limits, especially for models with billions of parameters and long-context training data. This paper introduces a suite of complementary techniques to reduce memory footprint without sacrificing model quality: (1) base model quantization with on-the-fly dequantization, (2) memory-efficient checkpointing combining selective activation caching and disk offloading, (3) softmax approximation using semantically relevant token subsets, and (4) logits masking. Experiments on Llama-3.2 3B and Qwen-2.5 3B demonstrate up to $26\times$ and $28\times$ reduction in peak memory, enabling fine-tuning on resource-constrained devices.

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **核心问题**：在边缘设备（如手机、IoT）上使用LoRA微调大语言模型（LLM）时，由于模型参数量大（数十亿）且长上下文训练数据导致激活存储爆炸，峰值内存远超消费级硬件限制，使得隐私保护、低延迟的个性化微调难以落地。
- **研究动机**：现有工作如QLoRA、梯度检查点等虽能降低内存，但激活内存仍为瓶颈；同时大多数方法依赖特定框架或需要足够CPU内存，不适用于资源极度受限的边缘设备。
- **整体含义**：本文提出一套互补技术，在不牺牲模型质量的前提下大幅降低峰值内存，从而在智能手机等边缘设备上实现隐私友好的LLM个性化微调。

## 2. 论文提出的方法论：核心思想与关键技术细节
论文包含四种互补技术，按内存瓶颈分析顺序叠加：

### (1) 基础模型量化与即时反量化
- **核心思想**：冻结的基础模型权重可被激进压缩。采用混合精度后训练量化：大部分线性层用INT4，输出头用INT8，输入嵌入用INT16，模型大小从5.98GB降至2.59GB。
- **关键技术**：为避免全精度权重常驻内存，提出“即时反量化”（Alg. 1）：前向和反向传播时从外存加载量化权重，临时反量化为FP32计算后立即丢弃，使得参数内存占用仅由量化权重本身决定（可离线存储）。

### (2) 内存高效检查点：选择性激活缓存与磁盘卸载
- **核心思想**：将网络分解为节点序列（嵌入层、n个解码器层、输出层），在反向传播时仅缓存边界激活，中间激活通过重计算获得；同时动态从磁盘加载权重，用完即弃。
- **关键技术**：Alg. 2 描述了三个阶段：第一阶段前向传播时，每个节点计算后立即将输出激活保存到外存并释放输入；第二阶段计算损失和LM head（输出层），该阶段不需要重计算，因而最大化节点大小以减少重计算开销；第三阶段反向传播时逐节点加载输入、重计算前向、计算梯度并释放。高峰内存出现在LM head阶段。

### (3) Softmax近似：基于语义相关的令牌子集
- **核心思想**：预训练模型的词嵌入空间具有语义几何性质——相似token的嵌入更接近。在微调中，仅需区分与目标token语义相近的少数负样本，因此可大幅缩小有效词汇表。
- **关键技术**：预计算所有token间的余弦相似度矩阵，对每个目标token取top‑k最相似索引，并通过查找表存储（仅INT32索引）。微调时，对一条序列的所有目标token取这些索引的并集构造子集J，仅加载J对应的嵌入子矩阵，计算缩减后的logits和softmax，减少I/O、内存和计算。图2显示top‑500效果接近全softmax。

### (4) Logits掩码
- **核心思想**：指令微调中大量非训练token（系统提示、上下文）不参与损失计算，传统做法仍计算其logits。本文提出直接跳过这些token的logits计算，使LM head处理的序列长度大幅缩短。
- **关键技术**：在损失函数中仅对训练token计算，非训练token的logits不生成，因此前向和反向的LM head计算仅涉及训练token，降低峰值内存。内存节省比例取决于训练token占比（XSum平均5.12%）。
